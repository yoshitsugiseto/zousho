# 蔵書管理システム - 書籍一覧の並び順変更とフィルタ追加の計画

## 背景・目的
ユーザーの利便性および管理者の運用効率向上のため、以下の変更を行います。
1. ダッシュボードの書籍一覧を「自分が借りている本 > 在庫ありの本 > 在庫なしの本」の順に並び替えます。
2. 管理者向けの書籍マスタ管理画面で、現在本を借りているユーザーによるフィルタリング機能（セレクトボックス）を追加し、さらに書籍一覧を「誰かに貸出中の本 > 貸し出していない本」の順に並び替えます。

## 変更内容

### Dashboard.tsx
#### `src/pages/Dashboard.tsx`
- **ソート機能の強化**
  - 現在のフロントエンドの表示用リスト `filteredBooks` に対し、マッピング後に `sort` メソッドを適用します。
  - ソートロジック：
    1. 自分が借りている本 (`myActiveLoans.length > 0`) が優先されます。
    2. 次に、在庫がある本 (`availableCount > 0`) が優先されます。
    3. 最後に、在庫がない（すべて誰かに借り入れられている等）本が表示されます。
    4. これらが同じ場合は、既存の通り生成日時（`created_at`）（Supabase側の基本並び順に準拠するか、タイトル名等でフォールバック）とします。

### AdminBooks.tsx
#### `src/pages/AdminBooks.tsx`
- **フィルタUIの追加**
  - stateとして `selectedUserId` (もしくは `selectedUserName`) を追加します（初期値は空文字列 または 'all'等）。
  - `books` から「現在貸出中のユーザー (activeLoans内のuser情報)」を抽出し、一意（ユニーク）なユーザーのリストを生成します。
  - 既存のテキスト検索入力欄の横、または下に「貸出中ユーザー」で絞り込むセレクトボックスを追加します。
- **フィルタロジックの適用**
  - `filteredBooks` の計算時に、上記セレクトボックスで選ばれたユーザーが一致する本のみを表示するように `filter` に条件を追加します。（未選択時は全表示）
- **ソート機能の強化**
  - フィルタ後のリストに対して `sort` メソッドを適用します。
  - ソートロジック：
    1. 貸出中の本 (`activeLoans.length > 0`) を優先します。
    2. 貸し出していない本 (`activeLoans.length === 0`) をあとにします。
    3. 上記が同じ場合は、生成時の基本順（Supabaseで `created_at` 降順にしているため、結果的に配列の元の順序を利用するか、ID等で安全にソート）にします。

## 検証計画

### 手動テスト
- `npm run dev` でローカルサーバーを起動して以下の確認を行います。
  - **一般ユーザーとしてログイン:**
    - ダッシュボードを開き、自身が借りている本が先頭に、続いて未貸出で在庫がある本、最後に在庫切れの本が並んでいることを目視確認します。
  - **管理者としてログイン:**
    - 書籍マスタ管理画面を開き、フィルター（セレクトボックス）が機能しているか確認します。
    - 貸出中の本がソート順の上位に表示されているか確認します。

### 自動テスト
- `npx vitest run` を実行し、既存コンポーネントのテスト (`__tests__/components/ProtectedLayout.test.tsx` 等) に波及エラーがないことを確認します。
- `npx playwright test` を実行し、`tests/dashboard.spec.ts` が引き続き通過することを確認します。
  - ※ ダッシュボードや書籍管理での要素の表示順が変わるだけであれば画面内表示そのものは壊れないため、既存テストのパスを主要な検証基準とします。
