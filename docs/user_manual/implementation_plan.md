# ユーザーマニュアルの実装計画 (Implementation Plan)

## Goal Description
アプリ内に非エンジニアでも直感的に理解できる丁寧な「ユーザーマニュアル」を組み込みます。ユーザーが迷わずに**招待メールからの初期設定**、**日々のログイン**、**蔵書の貸出・返却**を行えるように、アプリ内に専用のヘルプページを新設します。

## Proposed Changes

### フロントエンド（マニュアル画面の追加）
#### [NEW] `frontend/src/pages/Manual.tsx`
- Tailwind CSS を活用し、読みやすいレイアウト（目次つき、または機能ごとのカードレイアウト）でマニュアル画面を作成します。
- 記載する主なフロー：
  1. **はじめての方へ（初期設定）**
     - 招待メールのリンクをクリック
     - パスワードの設定
     - 2段階認証（Google Authenticator等）の設定とログイン
  2. **普段のログイン方法**
     - メールアドレスとパスワードの入力
     - 2段階認証コード（6桁）の入力
  3. **本の借り方・返し方**
     - ダッシュボードから本を探す方法
     - 「借りる」ボタンの操作と「自分の読書状況」の確認
     - 「返す」ボタンの操作
  4. **管理者向け機能（※該当者のみ）**
     - 書籍の追加・登録方法
     - ユーザーの招待・削除方法

### フロントエンド（ナビゲーションとルーティングの更新）
#### [MODIFY] `frontend/src/App.tsx`
- 新規ページ `<Route path="/manual" element={<Manual />} />` を追加します（ログインユーザー全員がアクセス可能）。

#### [MODIFY] `frontend/src/components/ProtectedLayout.tsx`
- ヘッダー部分（ナビゲーションバー）に「マニュアル（使い方）」へのリンクを追加し、いつでも参照できるようにします。

## Verification Plan

### 自動テスト / 静的解析
- TypeScriptのエラーがないこと、ルーティングが正常に追加されていることを確認します。

### 手動テスト (Manual Verification)
1. **ナビゲーションの確認**: ログイン後、ヘッダーから「マニュアル」画面へ正しく遷移できるか。
2. **マニュアル内容の視認性**: 非エンジニア視点で専門用語が使われていないか、手順がフロー通りに自然に読めるかを確認します。
3. **レスポンシブ崩れ**: PC、または狭い画面でもマニュアルが読みやすいようにレスポンシブ対応がされているかを確認します。
