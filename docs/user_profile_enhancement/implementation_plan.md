# ユーザープロフィール拡充とUI改善のプラン (Implementation Plan)

## Goal Description
ユーザー追加時の機能拡充と、アプリ内ヘッダー・管理画面のUI改善を行います。これにより、誰が本を借りているかや、現在ログインしているユーザーが誰かを直感的に把握できるようにします。

具体的には以下の3点に対応します。
1. **マニュアルの配置変更**: 画面ヘッダのマニュアルリンクを「ユーザー管理」の右側に移動します。
2. **招待時の名前登録と表示**: 管理者が新規ユーザーを招待する際に「名前（表示名）」も同時に入力できるようにし、ログインユーザーの名前がヘッダに表示されるようにします。
3. **書籍管理での名前表示**: 書籍管理画面において、貸出中のユーザーを「メールアドレス」ではなく「表示名（登録がない場合はメールアドレス）」で表示するように改善します。

## Proposed Changes

### バックエンドAPI (Edge Functions) の改修
#### [MODIFY] `supabase/functions/admin-user-management/index.ts`
- ユーザー招待API (`action === 'invite'`) において、リクエストボディから `email` に加えて `displayName` も受け取るように変更します。
- `supabase.auth.admin.inviteUserByEmail(email)` 実行後、生成されたユーザーのIDを用いて `public.users` テーブルの `display_name` を UPDATE する処理を追加します。

### フロントエンド（ユーザー招待・管理画面）
#### [MODIFY] `frontend/src/pages/AdminUsers.tsx`
- 新規ユーザー招待モーダルに「名前（表示名）」の入力フィールドを追加します。
- 招待実行時に、APIに `email` だけでなく `displayName` を送信するようにペイロードを変更します。

### フロントエンド（ヘッダー・レイアウト）
#### [MODIFY] `frontend/src/components/ProtectedLayout.tsx`
- ナビゲーションリンクの順序を変更します。「使い方（マニュアル）」のリンクを「ダッシュボード」の直後から、ナビゲーションリストの一番右（管理者の場合は「ユーザー管理」の右隣）に配置し直します。
- ※ヘッダの名前表示については、すでに `appUser?.display_name || session.user.email` を表示する仕組みが存在するため、データの紐付けさえできれば自動的に反映されます。

### フロントエンド（書籍管理画面での貸出中表示）
#### [MODIFY] `frontend/src/pages/AdminBooks.tsx`
- 書籍情報を取得する `fetchBooks` 関数において、貸出状況（`loans`）に紐づくユーザー情報として `email` に加えて `display_name` も取得するようにクエリを変更します（`.select('*, user:users(email, display_name)')`）。
- テーブルやカード内に「貸出中」ユーザーを表示している箇所を修正し、`user.display_name` が存在するならそれを優先的に表示するようにします。

## Verification Plan

### 手動テスト (Manual Verification)
1. **マニュアル配置の確認**: 一般ユーザーと管理者それぞれでログインし、マニュアルリンクが想定された右端（管理者の場合は「ユーザー管理」の右）に配置されているか確認します。
2. **名前登録とヘッダ表示の確認**: 管理者画面から「名前」を入力して新規ユーザーを招待し、そのユーザーでログインした際にヘッダに正しく「名前」が表示されるか確認します。
3. **貸出リストでの表示確認**: 登録したユーザーで本を借りた後、管理者の「書籍管理画面」において貸出中ユーザーが「名前」で表示されているかを確認します。
