# 実装計画: UI改善および招待メールの日本語化

## Goal Description
以下の3点の改善を実施し、ユーザー体験を向上させます。
1. **招待メールの日本語化**: Supabaseのダッシュボード設定を変更し、招待メールの文面を日本語化します。
2. **書籍検索フィルターの改善**: `Dashboard.tsx` および `AdminBooks.tsx` におけるタイトル・著者での検索において、アルファベットの大文字・小文字を区別せずにヒットするように修正します。
3. **招待モーダル文言の整理**: `AdminUsers.tsx` のユーザー招待モーダルにある「（ローカル環境の場合はInbucketにて確認できます）」という開発者向けの不要な文言を削除します。

さらに、ご提示いただいた書誌情報リストからISBNのみを抽出したファイルを作成いたします。

---

## User Review Required
> [!IMPORTANT]

> この手順につきましては、ユーザー様側でのダッシュボード操作をお願いする想定です。

## Proposed Changes

### 1. 招待メールの日本語化（Supabase設定手順）
Supabaseのダッシュボード（Authentication > Email Templates > Invite User）にて、以下のような日本語テンプレートへ手動で変更します。

**Subject (件名):**
```
Zousho（蔵書管理システム）へのご招待
```

**Message Body (本文 / HTML):**
```html
<h2>Zoushoへようこそ</h2>
<p>あなたはZousho（蔵書管理システム）のメンバーとして招待されました。</p>
<p>以下のリンクをクリックして、パスワードを設定しログインを完了してください：</p>
<p><a href="{{ .ConfirmationURL }}">招待を受け入れて設定を開始する</a></p>
```

### 2. 書籍検索フィルターの大文字小文字無視（Case-insensitive化）
#### [MODIFY] `frontend/src/pages/Dashboard.tsx`
#### [MODIFY] `frontend/src/pages/AdminBooks.tsx`
- フィルター処理部分の `includes` の呼び出し前に、検索文字列と対象文字列の両方を `.toLowerCase()` で小文字に変換して比較判定するように変更します。

### 3. モーダルの不要な文言削除
#### [MODIFY] `frontend/src/pages/AdminUsers.tsx`
- モーダル内の説明文である `入力したメールアドレス宛に、パスワード設定用の招待リンクが送信されます。（ローカル環境の場合はInbucketにて確認できます）` から、後半の括弧書きを削除します。

## Verification Plan

### Manual Verification
1. **検索の確認**: ダッシュボードおよび書籍管理画面で、「JavaScript」などのタイトルを持つ書籍に対し、「javascript」「JAVASCRIPT」などの小文字・大文字を混ぜた入力でも正しくヒットするか確認する。
2. **モーダルの確認**: 管理者画面から「ユーザーを招待」をクリックし、表記が「入力したメールアドレス宛に、パスワード設定用の招待リンクが送信されます。」のみになっているか確認する。
3. **メールの確認**: 実際に任意のメールアドレスへ招待を送信し、届いたメールの件名と本文が指定した日本語になっているか確認する。
