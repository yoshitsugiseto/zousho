# 実装計画: 書籍管理・貸出機能の権限別改善

ユーザー要望と現在のアプリの動作状況に基づき、役割（管理者・一般ユーザー）に応じた機能変更と不具合の修正を行います。

## User Review Required
> [!IMPORTANT]
> - 管理者による返却処理をどこで行うのが運用上スムーズかについて、今回は「書籍マスタ (`AdminBooks.tsx`) で各書籍の『貸出状況』を確認し、そこから返却ボタンを押す」形にて実装を進行しますが、よろしいでしょうか？

## Proposed Changes

### フロントエンド（Dashboard）
#### [MODIFY] [Dashboard.tsx](file:///Users/yseto/workspace/zousho/frontend/src/pages/Dashboard.tsx)
- **権限制御（一般ユーザー）**: 借入中であっても「本を返却する」アクションボタンは非表示にします。
- **権限制御（管理者）**: 管理者自身が本を借りる運用はないため、「本を借りる」アクションボタンを非表示とします。

### フロントエンド（AdminBooks - 管理画面）
#### [MODIFY] [AdminBooks.tsx](file:///Users/yseto/workspace/zousho/frontend/src/pages/AdminBooks.tsx)
- **バグ修正**: 「新しい書籍を登録」ボタン押下時に背景がグレイアウトし、フォーム操作ができない（CSSオーバーレイの競合）問題を修正します。モーダルコンテンツ側に `relative z-20` 等を付与します。
- **新規機能（編集）**: 書籍データに対して「編集」操作を追加します。タイトル・著者・総冊数を変更できるUIを設けます。
  - **総冊数の変更ロジック**: 
    - 増加する場合は、不足分の `book_items` を `status='available'` で新規追加（`insert`）します。
    - 減少する場合は、`status='available'` な `book_items` から不要分を削除（`delete`）します。（利用可能なアイテム数が足りない場合エラーを出してガードします）
- **新規機能（貸出管理・返却機能）**: 各書籍に紐づく現在貸出中のアイテム・ユーザーを確認するUIと、管理者として「返却処理」を行うボタンを追加します。

## Verification Plan

### Automated Tests
- `npm run dev` でローカルサーバーを起動し、Vite が正常にビルド・HMRすることを確認します。
- （Playwright等のE2Eテストについては、今回のUI変更が既存の `dashboard.spec.ts` などのシナリオに影響を与えるため、実装完了後にテストコード自体の修正が必要かを確認・相談します）

### Manual Verification
1. **管理者ログイン時**:
   - 書籍登録モーダルが正常に入力・操作できること。
   - 書籍情報の編集（タイトル、著者、および在庫の増減）が行えること。
   - 在庫現象時に、貸出中の本が誤って消去されないことのハンドリングができていること。
   - ダッシュボードで「借りる」ボタンが表示されないこと。
   - 書籍管理画面にて、貸出中の本を「返却」できること。
2. **一般ユーザーログイン時**:
   - ダッシュボードから本を借りられること。
   - 借りた後に「返却ボタン」が表示されない（または無効である）こと。
